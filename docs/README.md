# AI Operating Kit — Documentation

**AI is not the job. AI is the leverage layer.**

This directory contains the operating guides for the AI Operating Kit v1 — a lightweight, Markdown-first way for creative studios, game studios, executives, operators, and producers to run AI as a disciplined part of the business rather than a distraction from it.

These are prose guides, not code. Nothing here requires Node, Python, or npm. Read them, apply them, and adapt them to your own studio. The kit's skills (invoked by slash-name, e.g. `/spec-first`, `/roi-check`, `/wrapup`) turn the ideas below into repeatable moves.

## Suggested reading order

If you are a new executive or operator adopting the kit, read in this order:

1. **[survival-before-novelty.md](survival-before-novelty.md)** — Start here. It sets the priorities that make everything else meaningful. Without it, AI will happily help you do the wrong work faster.
2. **[ai-discipline-for-executives.md](ai-discipline-for-executives.md)** — The mindset shift: AI is leverage, not a second job. This reframes how you'll use every other guide.
3. **[avoiding-ai-side-quests.md](avoiding-ai-side-quests.md)** — The failure mode that costs the most time. Learn to spot it before it costs you a quarter.
4. **[executive-ai-workflow.md](executive-ai-workflow.md)** — Now make it a habit. A daily and weekly rhythm you can adopt this week.
5. **[model-routing-guide.md](model-routing-guide.md)** — Practical cost and quality control. Stop paying frontier prices for cheap-tier work.
6. **[ai-chain-of-custody.md](ai-chain-of-custody.md)** — Protect the studio's data, IP, and client trust before you scale usage.
7. **[studio-ai-governance.md](studio-ai-governance.md)** — Turn all of the above into a policy your whole team can follow.

Executives short on time can read (1), (2), and (4) and be operating well within a day. Studio leads responsible for teams and clients should not skip (6) and (7).

## How the docs relate to the skills

Each guide names the skills that make it real:

- **Plan and scope** — `/spec-first`, `/survival-filter`, `/roi-check`, `/forecast-check`
- **Execute and delegate** — `/model-router`, `/handoff-to-cowork`, `/task-ledger`
- **Protect and govern** — `/cleanroom`, `/decision-log`
- **Close and remember** — `/wrapup`, `/meeting-synthesis`, `/project-memory`

Read the docs for the reasoning. Use the skills for the reps.
