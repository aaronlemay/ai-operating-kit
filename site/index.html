01

What it is, and why it helps you

No Node, no Python, no npm, no accounts. Just a few Markdown files you drop into a project so any AI assistant — Claude, Claude Code, ChatGPT, Codex — works the same disciplined way every time.

Most people use AI reactively: they open a chat, start building, and three hours later have an impressive-looking thing nobody needed. The kit puts a thin layer of discipline in front of that impulse — spec before you build, check ROI before you spend, protect data before you paste, and preserve context so nothing gets lost between sessions.

It's built for operators: studio and production leaders, executives, and producers who need AI to be leverage on the real work, not a new full-time job.

Spec before you build

Force clarity on the problem, the user, and the smallest useful version before writing a line.

Kill the side quests

ROI and survival filters catch busywork before it eats a week.

Protect what matters

A cleanroom check guards client data, IP, and account boundaries.

Never lose context

Wrap-ups and project memory carry decisions across sessions and tools.

02

Start with the core four

Each “skill” is just a Markdown file with a copy-paste prompt. Open it, copy the prompt, paste it into your AI assistant. These four cover most of the value on day one.

/wrapup

Close cleanly

Condense a long session into a clean restart packet.

/spec-first

Clarity first

Define the problem before jumping into build mode.

/roi-check

Worth it?

Build now, prototype, defer, kill, or do it manually first.

/cleanroom

Protect data

Guard IP, client data, and account boundaries before pasting.

Pocket version

There are printable cut-out cards for these four in docs/checklist-cards.md — print it, keep it on your desk.

03

Set it up in about 15 minutes

Do this once on one real project. Understanding comes from using it, not from reading the whole kit first.

  1. Clone the repo

    Grab the kit from GitHub.

    git clone https://github.com/aaronlemay/ai-operating-kit.git
cd ai-operating-kit

  2. Install it into your project

    One command copies the skills, templates, and the CLAUDE.md operating rules into your project (and safely backs up any existing CLAUDE.md).

    ./scripts/install.sh /path/to/your/project

    Using ChatGPT instead? Paste CLAUDE.md into its Custom Instructions so the rules stay active.

  3. Start with /wrapup

    If you're mid-project, capture where you are first. Open skills/wrapup.md, copy the prompt, paste it in with your recent notes. You get back a clean restart packet — objective, status, decisions, risks, next actions.

  4. Run /spec-first on one real idea

    Pick something you're tempted to build. The skill forces out the smallest useful version and a do-not-build list. Most ideas shrink — that's the point.

  5. Pressure-test with /roi-check

    Paste your spec in and get a straight recommendation: build now, prototype, defer, kill, use an existing tool, or write a manual SOP first.

  6. Guard data with /cleanroom

    Before any client data, IP, or credentials touch an AI tool, run the checklist. It ends in a clear GO / NO-GO.

  7. Close with /task-ledger

    Turn everything you decided into a prioritized action list: Priority · Task · Owner · Status · Next Action · Blocker · Due.

The everyday flow

/spec-first → /roi-check → /cleanroom → build or hand off → /wrapup. Spec it, prove it's worth doing, protect the data, do the work, close cleanly.
