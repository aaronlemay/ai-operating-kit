# Templates

Fill-in-the-blank working documents you copy into a project and complete. Each one is Markdown-first, requires no Node/Python/npm, and is designed to be usable in minutes.

**Core idea:** AI is not the job. AI is the leverage layer. These templates give your work a stable structure so the AI (and your team) always has the context it needs.

## How to use

1. Copy the template file you need into your project (for example, into a `docs/` or `_ai/` folder).
2. Rename it if helpful (e.g. `project-brief-website-refresh.md`).
3. Replace every bracketed placeholder like `[Project name]`, `[Owner]`, `[YYYY-MM-DD]` with real content.
4. Delete the "How to use" note at the top once you have filled it in, or keep it as a reminder.
5. Keep the completed file in version control or your project memory so it survives across sessions and tools.

Templates are meant to be edited. Add or remove rows and sections as your project demands. Never leave placeholders in a document you hand to someone else.

## Which skill produces or consumes each template

Several skills read from or write to these formats. If you use the paired skill, it will fill or update the template for you; if you don't, the template stands alone.
